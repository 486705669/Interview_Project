# 老人陪伴助手：3 天落地执行方案（面试版）

## 1. 交付目标（与面试要求对齐）
1. 项目部署到服务器，手机浏览器可访问
2. 提交完整 PRD
3. 开发过程全程 Git，保留完整提交记录

## 2. 推荐实施策略
- 目标：先打通完整业务闭环，再做视觉与稳定性增强
- 原则：每天都有可运行版本，避免最后一天集成爆雷
- 优先级：应急模式 > 语音对话 > 提醒 > 记录查询 > 美化

## 3. 技术落地建议（可 3 天完成）
- 前端：Vue 3 + Vite + TypeScript + Pinia + Vue Router
- 后端：Express + TypeScript + Prisma
- 数据库：SQLite（开发）/ PostgreSQL（部署）
- 部署：Docker Compose + Nginx + HTTPS（Let's Encrypt）
- 语音：Web Speech API（先保证可用，再考虑替换云服务）

## 5. 三天排期（可直接照做）
## Day 1：打基础 + 主流程
1. 建仓与脚手架（前后端）
2. 完成老人端首页 + 对话页基本 UI
3. 接入语音输入/播报（可先浏览器原生）
4. 建立会话记录 API，落库对话日志
5. 完成紧急模式页面静态版（按钮 + 文案）

验收标准：
- 手机可访问并完成“说一句 -> 收到回答 -> 落日志”

## Day 2：提醒与应急闭环
1. 提醒 CRUD + 到点触发（前端轮询/后端定时）
2. 对话中高风险词识别，自动跳转紧急模式
3. 紧急事件日志落库（触发语句、动作、时间）
4. 家属/后台只读查询页（基础筛选）

验收标准：
- 可演示“高风险语句 -> 紧急模式 -> 事件可追溯”

## Day 3：部署、联调、演示收尾
1. Docker 化（前后端 + 数据库 + Nginx）
2. 云服务器部署，绑定域名并配置 HTTPS
3. 手机真机回归（语音权限、触控、字号）
4. 补齐 README、接口说明、演示脚本
5. 修复高优先级缺陷并冻结版本

验收标准：
- 外网可访问，面试现场可完整演示核心链路

## 6. Git 提交策略（示例）
建议每天至少 6~10 次小提交，保持可回滚与可审阅。

示例提交序列：
1. `chore: init monorepo structure and basic tooling`
2. `docs: add MVP PRD and delivery plan`
3. `feat(frontend): add mobile home and chat layout`
4. `feat(frontend): implement voice input and tts playback`
5. `feat(backend): add conversation api and persistence`
6. `feat(core): add emergency keyword detection`
7. `feat(frontend): add emergency mode page and actions`
8. `feat(reminder): add reminder create/list/ack flow`
9. `feat(admin): add family dashboard for event logs`
10. `chore(deploy): add docker compose and nginx config`
11. `docs: add deployment guide and demo script`
12. `fix: improve mobile interaction and error handling`

## 7. 部署步骤（简版）
1. 购买云服务器（2C2G 起步）
2. 安装 Docker / Docker Compose
3. 配置 `.env`（数据库连接、应用密钥、域名）
4. `docker compose up -d --build`
5. Nginx 反向代理前后端并配置 HTTPS
6. 手机访问域名，检查语音权限与接口连通

## 8. 演示脚本建议（面试高通过率）
1. 进入首页，语音问候并连续追问两轮（体现陪伴）
2. 新增“吃药提醒”，触发后点击“已完成”
3. 说出高风险语句，进入紧急模式并展示三按钮
4. 打开家属/后台页，展示该应急事件完整日志
5. 说明安全边界：不做诊断，优先求助
