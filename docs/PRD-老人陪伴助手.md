# 老人陪伴助手 PRD（MVP 版）

## 0. 文档信息
- 产品名称：老人陪伴助手
- 运行形式：手机 Web（浏览器访问）
- 交互方式：语音为主，文字为辅
- 文档版本：v1.0
- 编写日期：2026-03-05
- 目标交付周期：3 天（面试项目）

## 1. 产品概述
### 1.1 产品定位
让老人“开口就能用”的陪伴与生活助理：日常可稳定陪聊、可做基础生活提醒；遇到风险表达时，快速切入紧急引导流程并留痕，便于家属/后台追溯。

### 1.2 产品愿景
降低老年人独居时的孤独感与生活风险门槛，让语音成为最自然的交互方式，实现“像孙辈陪聊、关键时刻能帮忙”的体验。

### 1.3 目标用户
- 主要用户：60 岁以上老人（独居/半独居）
- 次要用户：家属（查看关键记录）
- 管理用户：后台运营/客服（查看会话与应急事件，做回访）

## 2. 用户痛点与需求映射
1. 陪伴不足
- 需求：低门槛聊天、主动回应情绪、可连续多轮对话

2. 记忆与执行力下降
- 需求：吃药/喝水/复诊提醒，提醒后可确认“已完成”

3. 信息获取门槛高
- 需求：语音问答，少输入、字大、路径短

4. 突发风险应对弱
- 需求：识别高风险表达并进入紧急模式，给出可执行步骤与联系人引导

5. 家属不放心且难追溯
- 需求：关键对话、提醒执行、应急记录可查询

## 3. 产品目标与成功指标
### 3.1 核心目标
- G1：语音对话可用且稳定，支持连续交流
- G2：应急场景有明确求助路径，优先安全
- G3：最小化用户画像可配置（偏好与紧急联系人）
- G4：形成关键记录，支持家属/后台查询

### 3.2 MVP 指标（上线后 1~2 周观测）
- 对话可用率（成功收音并返回回答）≥ 95%
- 首次响应时延（P95）≤ 3 秒
- 高风险表达命中后“进入紧急模式”触发率 ≥ 90%
- 提醒到点触达率 ≥ 95%
- 应急事件完整留痕率（时间+触发语句+处理动作）= 100%

## 4. 范围定义
### 4.1 MVP 范围（3 天内）
1. 语音对话（ASR + TTS）与文字兜底
2. 连续多轮对话（保存最近 N 轮上下文）
3. 提醒管理（新增、列表、到点提醒、确认完成）
4. 紧急模式（高风险词检测 + 求助路径引导）
5. 用户基础画像（昵称、常用称呼、兴趣、紧急联系人）
6. 记录中心（对话日志、提醒执行日志、应急事件日志）
7. 家属/后台只读查询页（账号简化为演示版）

### 4.2 非 MVP（本期不做）
- 医疗诊断与给药建议
- 视频通话、实时定位追踪
- 复杂权限体系与多角色审批流
- 深度个性化推荐算法

## 5. 角色与权限
### 5.1 老人端（手机 Web）
- 语音聊天、发文字、查看提醒、触发求助、管理基础信息

### 5.2 家属端（手机/PC Web）
- 查看关键记录（应急事件、提醒执行、近 7 天对话摘要）

### 5.3 后台运营端（PC Web）
- 检索用户、查看事件时间线、导出应急记录

## 6. 功能需求（详细）
## 6.1 F01 语音陪聊与问答
### 目标
让老人通过“按住说话/点击说话”即可完成聊天和简单问答。

### 功能点
1. 语音输入
- 点击麦克风开始录音，再次点击结束
- 显示“正在听”状态与波形动画
- 识别失败时提示“我没听清，再说一遍”

2. AI 回答
- 返回文字结果
- 自动语音播报（可关闭）
- 支持上下文连续追问（最近 5 轮）

3. 文字兜底
- 提供大字号输入框和常用快捷问题按钮

### 规则
- R1：语音优先，识别失败自动降级为文字输入引导
- R2：涉及医疗高风险问题不做诊断，触发安全话术与求助引导
- R3：敏感内容写入审计日志（仅后台可见）

## 6.2 F02 提醒助手
### 目标
帮助老人按时完成吃药/喝水/复诊等日常事项。

### 功能点
1. 新建提醒：标题、时间、重复规则（每天/每周）、备注
2. 到点提醒：弹窗 + 语音播报
3. 状态回执：已完成 / 稍后 10 分钟 / 今日跳过
4. 提醒历史：展示最近 30 天执行情况

### 规则
- R1：到点未响应 3 分钟，二次提醒
- R2：连续 2 次未响应，写入“低优先级关怀事件”供家属查看

## 6.3 F03 紧急模式（核心安全）
### 目标
识别潜在高风险表达并给出清晰可执行步骤。

### 触发方式
1. 用户主动点击“紧急求助”
2. 对话中命中高风险词（如“胸口痛得厉害”“呼吸困难”“晕倒”等）

### 紧急模式流程
1. 大字高对比界面，语音播报“先别慌，我现在帮你”
2. 立即展示三按钮：
- 拨打 120
- 联系紧急联系人
- 我还能说话（继续问答引导）
3. 自动记录触发时间、触发语句、用户点击动作
4. 若 30 秒无操作，重复语音提示并将事件标记为“待回访”

### 规则
- R1：优先引导求助，不输出诊断结论
- R2：应急日志不可删除，仅可追加备注
- R3：触发后会话顶部永久显示“紧急模式中”直到用户手动退出

## 6.4 F04 基础画像与偏好
### 字段
- 昵称、称呼偏好（如“王阿姨”）
- 兴趣标签（戏曲、新闻、健康操等）
- 紧急联系人姓名、关系、电话（必填）

### 规则
- R1：首次进入需完成最小信息（至少 1 个紧急联系人）
- R2：联系人电话格式校验；无效号码不允许保存

## 6.5 F05 记录中心（可追溯）
### 记录类型
1. 对话记录：时间、用户输入、系统回复、风险标签
2. 提醒记录：提醒内容、触发时间、用户回执状态
3. 应急记录：触发来源、原始语句、处置动作、结束时间

### 查询能力
- 老人端：仅可查看本人最近记录
- 家属/后台：按用户、日期、事件类型筛选

## 7. 用户流程
## 7.1 日常陪聊流程
1. 打开首页
2. 点击“按住说话”
3. 说出问题/聊天内容
4. 获取文字 + 语音回复
5. 连续追问
6. 结束会话，自动写入日志

## 7.2 提醒流程
1. 新建提醒
2. 到点弹窗 + 语音播报
3. 用户点“已完成/稍后”
4. 系统记录状态
5. 家属可查看执行历史

## 7.3 应急流程
1. 用户表达高风险症状或点“紧急求助”
2. 系统进入紧急模式
3. 用户选择拨打 120 或联系联系人
4. 系统记录全过程
5. 事件结束后可在后台追溯

## 8. 信息架构与核心页面设计
## 8.1 老人端页面
1. 首页（语音入口）
- 大号麦克风按钮
- 快捷入口：提醒、求助、历史
- 今日问候 + 最近提醒卡片

2. 对话页
- 聊天气泡（大字号）
- 麦克风主按钮 + 文字输入兜底
- 顶部“紧急求助”常驻按钮

3. 提醒页
- 提醒列表（按时间排序）
- 新建/编辑提醒弹层

4. 紧急模式页
- 红色高对比主题
- 三大按钮（120 / 联系家属 / 继续说）
- 当前时间与状态提示

5. 个人信息页
- 昵称、偏好、联系人管理

## 8.2 家属/后台页面
1. 用户列表页：按姓名/手机号检索
2. 事件时间线页：提醒、对话、应急统一时间轴
3. 应急详情页：触发语句、处置动作、处理结果备注

## 9. 交互与可用性规范（老年友好）
- 字号默认 18px 以上，关键按钮 20px 以上
- 主操作按钮高度不低于 48px
- 对比度遵循 WCAG AA
- 每个页面主任务不超过 3 个可点击重点项
- 所有关键提示支持语音播报
- 错误文案避免技术术语，使用“下一步怎么做”

## 10. 合规与安全边界
1. 明确声明：本产品不提供医疗诊断或替代医生建议
2. 高风险语句优先进入求助流程，不给治疗方案
3. 紧急联系人与日志数据加密存储
4. 日志访问最小权限控制（演示版可简化为账号隔离）
5. 用户可查看并导出自己的关键记录（演示版可提供截图导出）

## 11. 技术与数据方案（MVP）
## 11.1 建议技术栈
- 前端：Vue 3 + Vite + TypeScript + PWA（移动端优先）
- 后端：Node.js（NestJS/Express 均可）
- 数据库：PostgreSQL（或 SQLite 演示版）
- 语音能力：浏览器 Web Speech API（MVP）+ 可替换云 ASR/TTS
- 部署：1 台云服务器（Docker + Nginx + HTTPS）

## 11.2 核心数据表（建议）
1. users：用户基础信息
2. emergency_contacts：紧急联系人
3. conversations：会话主表
4. conversation_messages：会话消息
5. reminders：提醒任务
6. reminder_logs：提醒执行记录
7. emergency_events：应急事件

## 12. 验收标准（面试可演示）
1. 可在手机浏览器访问首页并正常语音交互
2. 可新增提醒并在模拟到点后完成回执
3. 输入高风险语句可进入紧急模式并展示求助按钮
4. 家属/后台可查到对应日志记录
5. 可演示“非医疗诊断”安全话术

## 13. 版本规划

- 完成 MVP 全链路演示（陪聊-提醒-应急-追溯）

### v1.1（可选加分）

## 14. 风险与应对
1. 语音识别不稳定
- 方案：文字兜底 + 重试提示 + 常用短语按钮

2. 应急误判/漏判
- 方案：关键词 + 规则双层触发；保留“手动紧急求助”常驻入口

3. 3 天开发时间紧
- 方案：先闭环核心链路，不做复杂权限与高阶美化
